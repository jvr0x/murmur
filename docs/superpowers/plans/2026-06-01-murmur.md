# Murmur Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A native macOS menu-bar app that records the mic while a hotkey is held, transcribes the speech via a local (or remote) STT server, optionally cleans it with an LLM, and inserts the text at the cursor.

**Architecture:** Swift Package executable (no Xcode project). An `NSApplication` accessory app drives a pipeline: `HotkeyManager` → `AudioRecorder` → `TranscriptionBackend` → optional `CleanupService` → `TextInserter`, coordinated by `DictationController` via an explicit state machine. STT/LLM are HTTP backends behind configurable base URLs. A `Scripts/make-app.sh` wraps the binary into a `Murmur.app` with an `Info.plist` so TCC permissions work.

**Tech Stack:** Swift 6.2 (language mode 5), SwiftPM, AppKit, AVFoundation, ApplicationServices/CoreGraphics (CGEventTap), SwiftUI (Settings/Onboarding via NSHostingView), whisper.cpp (sidecar), Ollama (OpenAI-compatible LLM).

**Design notes for consistency (used across tasks):**
- `AppConfig` property names are the single source of truth: `sttBackend` (enum `.whisperCpp`/`.openAICompatible`), `sttBaseURL: String`, `sttModel: String`, `language: String` ("auto" or code), `cleanupEnabled: Bool`, `llmBaseURL: String`, `llmModel: String`, `cleanupPrompt: String`, `cleanupTimeout: Double`, `hotkeyKeyCode: UInt16` (default 61 = Right Option), `insertionMethod` (enum `.paste`/`.keystroke`), `restoreClipboard: Bool`, `whisperServerPort: Int`.
- STT result type: `struct Transcript { let text: String }`.
- Backend protocol: `protocol TranscriptionBackend { func transcribe(wav: Data, language: String) async throws -> Transcript }`.
- All UI-touching types are `@MainActor`. Networking uses `URLSession` `async` APIs.
